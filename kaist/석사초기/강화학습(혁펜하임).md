$\epsilon$이 각 episode에 따라 점차 감소하는 방식

**Discount factor $\gamma$(0~1)**
미래에 받을 모든 보상을 현재 가치로 환산하기 위해 계속해서 곱해나가는 비율
![[Pasted image 20260303120906.png|250]]
- **$\gamma$가 작을 때 (예: 0.1):** 미래에 큰 보상이 기다리고 있어도, $\gamma^2, \gamma^3$으로 갈수록 값이 0에 매우 빠르게 수렴 즉, 에이전트는 "먼 미래의 큰 보상보다 당장 1~2스텝 뒤에 받을 작은 보상이 훨씬 중요해!"라고 판단=>현재에 더 중점
- **$\gamma$가 클 때 (예: 0.99):** 미래의 보상도 현재와 거의 비슷한 가치를 가집니다. 에이전트는 "지금 당장은 보상이 없어도, 긴 경로를 통해 얻을 큰 미래의 보상을 위해 참자!"라고 판단=>미래에 더 중점
**Q-update**
다음과 같은 식을 통해 update
$$Q(s_{t},a_{t}) \leftarrow (1-\alpha)Q(s_{t},a_{t})+\alpha (R_{t}+\gamma \cdot maxQ(s_{t+1},a_{t+1}))$$
$0<\alpha<1$ => 새로운 정보를 얼마나 받아들일 것이냐


# MDP (Markov Decision(action) Process)
![[Pasted image 20260303125332.png|300]]
Goal= maximize **expected return**
return $G_{t}\overset{\Delta}{=}R_{t}+\gamma R_{t+1}+\gamma^{2}R_{t+2}+...$
maximize하는 policy를 찾는 것
Policy: $P(a_{t}|s_{t})$
Transition probablity: $P(s_{t+1}|s_{t},a_{t})$

**MDP를 구성하는 5요소 (S, A, P, R, $\gamma$)**
- **State ($S$):** 에이전트가 관찰할 수 있는 **상태**의 집합 (예: 바둑판의 돌 배치, 로봇의 현재 위치)
    
- **Action ($A$):** 에이전트가 할 수 있는 **행동**의 집합 (예: 상/하/좌/우 이동, 바둑돌 놓기)
    
- **Transition Probability ($P$):** 어떤 상태에서 행동을 했을 때, **다음 상태로 갈 확률** (환경의 역학) (불 확실성의 확률)
    
- **Reward ($R$):** 특정 행동을 했을 때 환경이 주는 **보상** (피드백)
    
- **Discount Factor ($\gamma$):** 미래 보상을 얼마나 중요하게 여길지 결정하는 **감가율**

**Markov** => 미래는 오직 현재에     의해서만 결정된다. (과거의 데이터는 신경쓰지 않고, 오직 현 state에서 다음 action을 판단)
### Policy와 Transition Probability 차이
#### 2️⃣ **Policy (정책)**: 운전자가 **어떤 방향으로 갈지 결정하는 방법**

- 예를 들어, "오른쪽으로 가는 확률 70%, 왼쪽으로 가는 확률 30%"
- 즉, 내비게이션(혹은 네가) 결정하는 행동 선택 전략
- 운전자가 “어느 길로 갈지” 정하는 확률 분포야.
    greedy인지 $\epsilon$-greedy인지가 결국 policy(정책) 만약 greedy라면, 확률이 1이 된다는것.

---

#### 3️⃣ **Transition Probability (전이확률)**: 도로와 교통 상황이 결정하는 **운전을 했을 때 실제 결과**

- 예를 들어, 오른쪽으로 돌았는데:
    
    - 80% 확률로 정상적으로 ‘B 교차로’에 도착
        
    - 20% 확률로 공사 중이라 ‘A 교차로’에 다시 돌아옴
        
- 즉, 네가 선택한 행동(우회전)을 했을 때 환경이 주는 결과 확률이야.
# **optimal policy**
**state value function (V)** (상태 가치 함수) : **지금**으로부터 **(현재 state)** 기대되는 return
$$V(s_{t})\overset{\Delta}{=}\int G_{t}\,P(a_{t},s_{t+1},a_{t+1},...|s_{t} )\,d\,a_{t}:a_{\infty}$$
=> $V(s_{t})$를 maximize하는 $P(a_{t}|s_{t}),\, P(a_{t+1}|s_{t+1}), P(a_{t+2}|s_{t+2})...$ 가 optimal policy

**action value finction (Q)** (행동 가치 함수) : **지금 행동**으로부터 기대되는 return
$$Q(s_{t},a_{t})\overset{\Delta}{=}\int G_{t}\,P(s_{t+1},a_{t+1},s_{t+2},a_{t+2},...|s_{t},a_{t})\,ds_{t+1}:a_{\infty}$$
**Bellman equation**
state/action value function을 재귀적으로 표현하는 방법
-V를 Q를 통해 표현
$$V(s_{t})\overset{\Delta}{=}\int G_{t}\,P(a_{t},s_{t+1},a_{t+1},...|s_{t} )\,d\,a_{t}:a_{\infty}=\int\int G_{t}P(s_{t+1},a_{t+1},...|a_{t},s_{t} )\,d\,s_{t+1}:a_{\infty}\;P(a_{t}|s_{t})da_{t}$$
$$V(s_{t})\overset{\Delta}{=}\int Q(s_{t},a_{t})P(a_{t}|s_{t})da_{t}$$
=> 현 state에서 모든 action에 대한 평균값을 취한 값

-V를 next V를 통해 표현
$$G_{t}=R_{t}+\gamma G_{t}$$
$$V(s_{t})\overset{\Delta}{=}\int(R_{t}+\gamma V(s_{t+1}))P(a_{t},s_{t+1}|s_{t})\;da_{t},s_{t+1}$$
$P(a_{t},s_{t+1}|s_{t})=P(s_{t+1}|a_{t},s_{t})P(a_{t}|s_{t})$   =>Transition probability와 Policy의 결합 형태-> State value function ($V(s_{t})$)를 maximize 할 Policy를 찾아 주는 것.

-Q를 next V를 통해 / Q를 next Q를 통해...

![[Pasted image 20260304004435.png|697]]
Action value function ($Q(s_{t},a_{t})$)를 쪼개보면 위 이미지(파란원)처럼 Policy와 Transition Probability의 연속으로 나타나게 됨 => Q가 maximize(Q*) 될 수 있는 policy들을 P* 로 표현하고, $s_{t}$는 정해져 있는 상태이기에, Q를 $a_{t}$에 대한 그래프로 그려볼 수 있음=> Q가 max가 되는 곳에 $a_{t}$를 max로 찍어주면 ($\delta$ function을 통해) 결국 state value function이 max가 될 것임--> 이것이 결국 $\epsilon$ greedy action.

그렇다면 결국 우리의 목표는 Q* 를 찾는것.

### Monte Carlo
Monte Carlo: 기대값 계산이 까다로울 때 sampling을 통해 numerical하게 approximate하는 방식 (큰수의 법칙)
$$E[x]\overset{\Delta}{=}\int xP(x)\,dx\approx \frac{1}{N}\sum^{N}_{i=1}x_{i}$$
이를 통해 action value function을 다음과 같이 표현
$$Q(s_{t},a_{t})\overset{\Delta}{=}\int G_{t}\,P(s_{t+1},a_{t+1},s_{t+2},...|s_{t},a_{t})\,ds_{t+1}:a_{\infty}\approx \frac{1}{N}\sum^{N}_{i=1}G^{(i)}_{t}$$
![[Pasted image 20260304173128.png]]
현재 선택한 정책(policy)에 대해서 얻을 수 있는 value를 평균적으로 구하기 (Q업데이트와 적분은 별개라는 것을 기억. 적분은 이론적 정의 일뿐)
### Temporal difference
$$Q(s_{t},a_{t})=\int (R_{t}+\gamma Q(s_{t+1},a_{t+1}))P(s_{t+1}|s_{t},a_{t})P(a_{t+1}|s_{t+1})\; ds_{t+1},a_{t+1}\approx \frac{1}{N}\sum^{N}_{i=1}(R^{(i)}_{t}+\gamma Q(s^{(i)}_{t+1},a^{(i)}_{t+1}))$$
$$\frac{1}{N}\sum^{N}_{i=1}(R^{(i)}_{t}+\gamma Q(s^{(i)}_{t+1},a^{(i)}_{t+1}))\overset{\Delta}{=} \bar{Q}_{N}=\frac{1}{N}(\bar{Q}_{N-1}(N-1)+R^{(N)}_{t}+\gamma Q(s^{(N)}_{t+1},a^{(N)}_{t+1}))$$
((N-1번째 기준 평균) X (N-1)+N번째 sample)
$$Q(s_{t},a_{t})=\bar{Q}_{N-1}+\frac{1}{N}(R^{(N)}_{t}+\gamma Q(s^{(N)}_{t+1},a^{(N)}_{t+1})-\bar{Q}_{N-1})$$
=> **1 step TD**
현 state 기준 다음 state에서 일어나는 action value를 통해 현 state의 action value function update
$$\begin{aligned}\bar{Q}_{N}&=\bar{Q}_{N-1}+\frac{1}{N}(R^{(N)}_{t}+\gamma Q(s^{(N)}_{t+1},a^{(N)}_{t+1})-\bar{Q}_{N-1})\\&=(1-\alpha)\bar{Q}_{N-1}+\alpha(R^{(N)}_{t}+\gamma Q(s^{(N)}_{t+1},a^{(N)}_{t+1}))\end{aligned}$$
=>기존 Q-learning과 비슷한 형태, but Q의 max를 취하는 것은 아님=>위 방법이 **SARSA**
(위 처럼 다채로운 sample을 얻기 위해 decaying $\epsilon$-greedy 사용, action value function Q는 현 state에서 특정 action에 대한 return 기대값을 나타내는 local value임을 기억)

## On/Off-policy
Behavior Policy (행동 정책): 실제로 환경(Environment)과 상호작용하며 **데이터를 수집하는 정책**
Target Policy (대상 정책): 업데이트의 기준이 되는 **학습의 목표 정책**

On-policy : behavior policy=target policy
Off-policy : behavior policy$\neq$target policy

=> $s_{t}$단계에서 내가 선택한 $a_{t}$에 대해 $R_{t}$와 다음 step의 $s_{t+1}$에서의 $a_{t+1}$를 판단하고 이를 통해 $Q(s_{t}, a_{t})$를 업데이트. 하지만, $s_{t+1}$에서 실제로 행하는 action $q_{t+1}$은 앞선 판단된 action $a_{t+1}$과는 다른 action $q_{t+1}$이 될 것
-> 내가 update한 Q는 내가 임의로 판단한 action $a_{t+1}$에 대해 update된 상태인거고, 만약 다음 episode에서 같은 step $s_{t}$가 발생하였을 때, 이전 episode에서 추가로 update된 Q를 통해 더 나은 $a_{t+1}$를 판단  
(On-policy는 $a_{t+1}=q_{t+1}$이 되는거라 생각)

Off-policy의 장점
1. 사람의 학습 방식을 모방 (이전의 실수에 대해 더 나은 판단을 내릴 수 있었음을 깨닫는?)
2. 여러번 탐험 하면서 optimal policy ($\epsilon$)
3. 재평가 가능

**다시 보자 Q-learning**

Target : (greedy)      $\begin{aligned}&P(a_{t+1}|s_{t+1}) =\delta (a_{t+1}-a^{*}_{t+1}),\\&a^*_{t+1}=argmax_{a_{t+1}}Q(s_{t+1},a_{t+1})\end{aligned}$

Behavier: $\epsilon$-greedy
$$\begin{aligned}Q(s_{t},a_{t})&=\int(R_{t}+\gamma Q(s_{t+1},a_{t+1}))P(a_{t+1}|s_{t+1})P(s_{t+1}|s_{t},a_{t}))ds_{t+1},\,a_{t+1}\\
&=\int(R_{t}+\gamma Q(s_{t+1},a_{t+1}))\delta (a_{t+1}-a^*_{t+1})P(s_{t+1}|s_{t},a_{t}))ds_{t+1},\,a_{t+1}\\
&=\int(R_{t}+\gamma \underset{a_{t+1}}{max}Q(s_{t+1},a_{t+1}))P(s_{t+1}|s_{t},a_{t}))ds_{t+1}=>R_{t}+\gamma\, \underset{a_{t+1}}{max}Q(s_{t+1},a_{t+1})
\end{aligned}
$$

$R_{t}+\gamma\, \underset{a_{t+1}}{max}Q(s_{t+1},a_{t+1})$=> TD-target!

**Q-learning vs SARSA**

위 그림에서, ((2,2) state)에서 움직인다 할 때, 오른쪽-아래로 움직이게 될 경우 SARSA는 오른쪽 방향에 대한 값을 -99(대략)으로 잡아버리게 됨.-> 비효율 적으로 돌아가게 된다(겁쟁이).

#### n-step TD& n-step Q-learning
$$\begin{aligned}G_{t}&=R_{t}+\gamma R_{t+1}+\gamma^{2} R_{t+2}...\\
&=R_{t}+\gamma G_{t+1} &\rightarrow one\;step \\
&=R_{t}+\gamma R_{t+1}+\gamma^{2}G_{t+2} &\rightarrow two\;step
\end{aligned}
$$
return을 위처럼 2 step 다음까지 고려해서 하면 2-step
![[Pasted image 20260305175218.png]]
다만 이렇게 되면 next-next step에 대한 정보가 필요하다, 즉 t+1번째 step에서도 behavior policy (q)를 따를 수 밖에 없게 된다. 이를 해결하기 위한 방법=> **importance sampling** 
importance sampling:
$$\begin{aligned}&\int xP(x)dx\approx\frac{1}{N}\sum^{N}_{i=1}(x_{i})\\
=&\int x\frac{P(x)}{q(x)}q(x)dx\approx \frac{1}{N}\sum^{N}_{i=1}x_{i}\frac{P(x_{i})}{q(x_{i})}
\end{aligned}$$
위 과정을 통해 $q(a_{t+1}|s_{t+1})$를 추출하여 t+1 step에서 action 진행. 이렇게 되면 target policy는  
$$Target:\frac{P(a_{t+1}|s_{t+1})}{q(a_{t+1}|s_{t+1})}(R_{t}+\gamma R_{t+1}+ \gamma^2\underset{a_{t+2}}{max}Q(s_{t+2},a_{t+2}))$$
가 됨. 하지만 $\frac{P(a_{t+1}|s_{t+1})}{q(a_{t+1}|s_{t+1})}$는 보통 사용 x (target policy P는 $\delta$ function이라 0 또는 $\infty$라.)
